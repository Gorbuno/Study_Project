Реализовать алгоритмы NFDH и FFDH для поиска решений упаковки прямоугольных объектов, заданных в файле “ads_lab6_rnd.csv” (описание формата приведено далее) , на ленте шириной 1000 условных единиц. Оценить степень приближения решения к оптимальному.

В файле “ads_lab6_rnd.csv” приведено 133 прямоугольных объекта, которые исходно являлись компонентами случайно произведенного разреза квадрата размером 1000×1000 условных единиц. Таким образом, существует оптимальное размещение их на ленте шириной в 1000 единиц, которое будет занимать ровно 1000 условных единиц высоты. Однако, найти это оптимальное расположение является вычислительно сложной задачей, и в ходе данной работы целью будет оценить, насколько алгоритмы NFDH и FFDH приблизятся к оптимальной высоте в 1000 единиц.

Формат файла для выполнения задания

Файл “ads_lab6_rnd.csv” содержит 133 строки (символ окончания строки x0Dx0A)
В каждой строке (строки заголовка – нет) записаны сведения о прямоугольных объектах – два числа (разделитель – точка с запятой):
	ширина объекта: целое число из диапазона 1…1000
	высота объекта: целое число из диапазона 1…1000

Пример первых нескольких строк файла:

134;76
115;85
10;4
14;82
. . .



Программа должна вывести на экран:
	для каждого ряда:
o	порядковый номер (считая снизу)
o	высоту
o	через запятую – список объектов, входящих в ряд (достаточно выводить ширину объекта)
o	ширину оставшейся незаполненной части ряда
	итоговую высоту размещения на ленте (сумму высот всех рядов)
